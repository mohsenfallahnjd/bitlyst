# Promise APIs Comparison: `all` vs `race` vs `any` vs `allSettled`

JavaScript provides multiple Promise combinators.  
They all **start promises concurrently**, but they differ in **when they resolve/reject** and **what they return**.

Understanding the differences prevents subtle bugs and improves performance.

---

## рҹ§  Quick Overview

| API | Resolves when | Rejects when | Typical use |
|----|--------------|--------------|-------------|
| `Promise.all` | All promises resolve | Any promise rejects | All-or-nothing |
| `Promise.race` | First promise settles | First promise settles | Timeout / fastest |
| `Promise.any` | First promise resolves | All promises reject | First success |
| `Promise.allSettled` | All promises settle | вқҢ Never rejects | Collect results |

---

## рҹ”№ Promise.all

### Behavior
- Waits for **all promises to resolve**
- Rejects immediately if **any promise rejects**

```js
await Promise.all([
  fetch("/a"),
  fetch("/b"),
  fetch("/c"),
]);
```

### Timeline
```

C в”Җв”Җв”Җв”Җв”Җв”ҳ
```

### When to use
вң… You need **all results**  
вқҢ One failure should stop everything

### Common mistake
```js
await Promise.all(tasks); // crashes if one fails
```
If partial success is acceptable вҶ’ use `allSettled`.

## рҹ”№ Promise.race

### Behavior
- Settles as soon as **the first promise settles**
- That settlement can be **resolve or reject**

```js

  fetch("/data"),
  timeout(3000),
]);
```

### Timeline

### When to use
вң… Timeouts  
вң… First response wins  
вқҢ Not for вҖңfirst success onlyвҖқ

## рҹ”№ Promise.any

### Behavior
- Resolves on the **first fulfilled promise**
- Rejects only if **all promises reject**
- Rejection reason is an `AggregateError`

```js
const result = await Promise.any([
  fetch("/cdn-1"),
  fetch("/cdn-2"),
  fetch("/cdn-3"),
]);
```

### Timeline
```
cdn-1 вң–
cdn-2 вң–
cdn-3 вң” в”Җв”Җв–ә resolve
```

### When to use
вң… Redundant endpoints  
вң… Fallback strategies  
вқҢ When failures matter

---

## рҹ”№ Promise.allSettled

### Behavior
- Waits for **all promises to settle**
- Never rejects
- Returns status for each promise

```js
const results = await Promise.allSettled([
  fetch("/a"),
  fetch("/b"),
  fetch("/c"),
]);
```

### Result shape
```js
[
  { status: "fulfilled", value: ... },
  { status: "rejected", reason: ... },
]
```

### Timeline
```
A вң”
B вң–
C вң” в”Җв”Җв–ә resolve with report
```

### When to use
вң… Logging  
вң… Batch operations  
вң… Partial success allowed

## рҹ§  Mental Model

Think of them as **rules for waiting**:

- `all` вҶ’ everyone must succeed
- `race` вҶ’ whoever finishes first decides
- `any` вҶ’ first success wins
- `allSettled` вҶ’ everyone reports back

---

## вҡ пёҸ Important Notes

- All promises **start immediately**
- None of these APIs make JS parallel
- They only coordinate async results

This is still sequential:
```js
await a();
await b();
```

This is concurrent:
```js
await Promise.all([a(), b()]);
```
